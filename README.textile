h1. AttributeRouting 

_Define your routes using attributes on action methods in ASP.NET MVC._

h2. Overview

Use RouteAttributes to decorate actions in your controllers. There is a specific RouteAtribute for each of the four HTTP verbs: GET, PUT, POST, and DELETE.

Below is an example of defining routes for a resource in a RESTful style. Note that all four verbs are represented.

<pre>
    <code>
        public class ResourcesController : ControllerBase
        {
            [GET("Resources")]
            [GET("Resources/Index")]
            public ActionResult Index()
            {
                return View();
            }

            [GET("Resources/New")]
            public ActionResult New()
            {
                return View();
            }

            [PUT("Resources")]
            public ActionResult Create()
            {
                Flash("Resource Created");
                return RedirectToAction("show", new { id = 1 });
            }

            [GET("Resources/{id}")]
            public ActionResult Show(int id)
            {
                return View();
            }

            [GET("Resources/{id}/Edit")]
            public ActionResult Edit(int id)
            {
                return View();
            }

            [POST("Resources/{id}")]
            public ActionResult Update(int id)
            {
                Flash("Resource Updated");
                return RedirectToAction("show");
            }

            [GET("Resources/{id}/Delete")]
            public ActionResult Delete(int id)
            {
                return View();
            }

            [DELETE("Resources/{id}")]
            public ActionResult Destroy(int id)
            {
                Flash("Resource Destroyed");
                return RedirectToAction("index");
            }
        }
    </code>
</pre>

Once you've added the RouteAttributes to your action methods, you will need to have the AttributeRouting library find them and register  the corresponding routes. You can do this in your Global.asax:

<pre>
    <code>
        public static void RegisterRoutes(RouteCollection routes)
        {
            routes.IgnoreRoute("{resource}.axd/{*pathInfo}");

            routes.MapAttributeRoutes();
        }
    </code>
</pre>

When you're done, hit F5. That's it.

h2. More Advanced Usage

The AttributeRouting.Web project's home page has complete documentation on how to use all of the features available. 

Briefly, you can:
* define areas,
* define nested routes,
* define multiple routes that map to a single action,
* add route defaults and constraints, and
* specify route precedence,

In addition, there are html helpers for generating RESTful forms and a few other handy bits.
